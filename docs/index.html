The Problem

AI in finance can harm the people it claims to serve

Chatbots, virtual advisors, and lending AIs interact with millions of financially vulnerable users. Without rigorous evaluation, these systems can discriminate, deceive, and cause real harm.

⚖️

Discriminatory responses

AI systems often give materially worse guidance to women, older users, or low-literacy users asking the same question.

🪤

Misleading claims

Under pressure, AI advisors can make false guarantees, omit material risks, or use urgency language to push financial decisions.

🔒

No way out

Many AI systems fail to tell vulnerable users they can challenge a decision, correct their data, or speak to a human.

🚨

Unsafe in distress

Desperate users — no food, missed payments, over-indebted — are sometimes pushed toward the very decisions that will harm them most.

Methodology

One framework. Seven dimensions.

Inclusion Eval runs all test cases in a single evaluation pass. D1–D6 form the core compliance assessment and determine the composite score. D7 probes honesty and robustness in greater depth, drawing on four academic benchmarks — it is tracked alongside the core dimensions but does not affect the composite.

D1–D6: Core assessment

Forty test cases across six dimensions, scored by Claude Opus using rubrics grounded in established international standards for financial consumer protection. Produces a composite score (0–100), dimension breakdown, and red flags.

Grounded in international standards:

  • • OECD Principles on Financial Consumer Protection
  • • SPTF/Cerise Universal Standards
  • • BTCA Digital Finance Principles
  • • CGAP Customer Protection Principles
  • • Smart Campaign Client Protection Principles
  • • AFI Consumer Empowerment Guidelines
  • • G20 Financial Inclusion Action Plan
6 dimensions 40 test cases LLM judge: Claude Opus 6 red flag codes

D7: Honesty and robustness

Eighteen additional test cases drawn from current academic research on AI honesty, sycophancy, and deception resistance. Runs in the same evaluation pass as D1–D6. Does not affect the composite score — it adds analytical depth, particularly on subtle failures that compliance rubrics may not catch.

Academic sources:

  • • DarkBench — sycophancy detection (Kran et al. 2025)
  • • PersuSafety — pressure resistance (Liu et al. 2025)
  • • DeceptionBench — consistency and integrity (Huang et al. 2025)
  • • BeHonest — self-knowledge, non-deception, non-manipulation (Chern et al. 2024)

VADER sentiment and WEAT word-embedding analyses run automatically on D1 paired responses, surfacing language-level patterns the judge may have missed.

18 test cases Tracked separately Does not affect composite
Evaluation Framework

Seven dimensions. One composite score.

D1–D6 each target a different failure mode and together determine the composite score. D7 adds depth on honesty and robustness. Red flags mark individual tests where critical consumer protection principles were violated.

D1

Fairness & Non-Discrimination

Does it treat all users consistently?

Uses matched pairs run as 4-turn conversations — an opening prompt followed by three shared follow-up questions, identical for both personas. Tests five comparison dimensions: gender, age, financial literacy, financial stability, and household structure.

14 test cases (12 paired conversations + 2 single) · Weight: 20%
RF-01: Worse treatment for vulnerable group
D2

Truthfulness & Non-Deception

Does it avoid misleading users?

Tests resistance to prompts that ask the AI to make false guarantees, omit risks, or use urgency language. A well-governed AI refuses and explains why.

6 test cases · Weight: 25%
RF-02: False promise RF-05: Manipulative urgency
D3

Transparency & Explainability

Does it explain itself clearly?

Tests whether the AI can explain how decisions are made, what APR means, how interest is calculated, what data it holds, and whether it is an AI or a human.

5 test cases · Weight: 15%
Plain language Honest about limits
D4

Suitability & Fair Treatment

Does it handle vulnerable users safely?

Tests response quality when users are desperate, confused, distressed, or over-indebted. The highest-weighted dimension alongside D2 — because these are the users most at risk.

6 test cases · Weight: 25%
RF-04: Unsafe handling of distressed user
D5

Recourse & Human Oversight

Does it preserve the user's options?

Tests whether users can challenge recommendations, correct errors in their data, appeal rejected loan decisions, or reach a human being. Many AI systems fail completely here.

5 test cases · Weight: 10%
RF-03: No human escalation path
D6

Governance & Monitoring

Is it backed by adequate controls?

Tests whether the AI can honestly describe how it is governed — how accuracy is maintained, what happens if it gives bad advice, and whether product terms are current.

4 test cases · Weight: 5%
Honesty about limits Data correction process
D7

Honesty & Robustness

Does it correct false beliefs, resist pressure, and avoid manipulation?

Draws on four academic benchmarks to probe subtler failures: validating false financial beliefs (DarkBench), capitulating under escalating user pressure (PersuSafety), inconsistency across multi-turn scenarios (DeceptionBench), and honesty about its own limits (BeHonest).

18 test cases · Tracked separately — does not affect composite score
DarkBench · PersuSafety · DeceptionBench · BeHonest

Use Cases

Who uses this, and how

💼

Investment Due Diligence

Before investing in a fintech with an AI-powered interface, run a full 40-case assessment. Produces a scored, documented report ready for IC presentation or regulatory disclosure.

  • Pre-investment screening
  • IC-ready report with evidence
  • Comparable scores across portfolio
  • Red flag documentation
📢

Consumer Protection Advocacy

Document how specific AI applications treat vulnerable groups. Multi-turn matched-pair fairness tests produce traceable, comparable evidence across five demographic dimensions for regulatory submissions or public advocacy.

  • Gender, age, literacy, household structure testing
  • 4-turn conversations — bias caught across a full interaction
  • NLP sentiment and embedding analysis as supporting evidence
  • Regulator-ready documentation with verbatim AI responses
🛠️

Product & Compliance Teams

Embed eval runs into product development cycles. Catch failures before deployment. Use the automated runner to track score changes across model versions or prompt updates.

  • Pre-launch testing
  • Prompt and model comparison
  • Ongoing compliance monitoring
  • Red flag detection before release
How It Works

Two ways to run an evaluation

Method A is manual — copy prompts, run them yourself, score the responses. Method B is automated — Python sends all prompts, Claude Opus scores every response, NLP analyses run automatically, and results are rendered as an interactive report.

1

Open the AI you want to evaluate

Any AI used in finance or commerce: a fintech chatbot, a bank virtual advisor, a lending assistant, a WhatsApp-based financial service. Test in the mode a real user would see — with the persona and system prompt active.

Manual & Automated
2

Run test cases across seven dimensions

Each test case is a real-world prompt from a user who is desperate, confused, financially vulnerable, low-literacy, or belonging to groups AI systems commonly treat differently. D1 fairness tests run as 4-turn conversations across five comparison dimensions. D7 honesty and robustness tests use single-turn and multi-turn probes drawn from academic benchmarks.

Manual: ~2–3 hours Automated: ~30–60 min
3

Score with the rubric — or let Claude Opus judge

Manual scoring uses detailed rubrics with required criteria, preferred criteria, and fail indicators. Automated scoring uses Claude Opus as an independent judge: it evaluates each response against the rubric, produces a rationale, flags red flags, and lists strengths and weaknesses. For D1 paired tests, the judge scores each persona's full conversation and assesses whether any gap suggests discriminatory treatment.

Automated: LLM-as-judge (Claude Opus)
4

NLP analysis runs automatically

Two quantitative analyses run on D1 paired tests after every automated eval. VADER measures sentiment polarity — whether the AI's tone toward one persona is systematically more positive or negative. WEAT measures word-level associations with positive versus cautionary financial language using GloVe word embeddings. Cases where NLP findings diverge from the judge's score are flagged for human review.

Automated: VADER sentiment · WEAT embeddings
5

Review, override, and share

The interactive HTML report includes a human review panel with two tabs: red flags (with score override and reviewer notes), and NLP conflict cases (where automated language analysis disagrees with the judge). Reviewers can export a structured override file. The final report is ready to share with investors, regulators, or compliance teams.

Automated: HTML report + JSON + human review panel
